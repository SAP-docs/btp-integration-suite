<td valign="top">

Extended Handling for UN/EDIFACT Segment Groups During MIG Migration

</td>
<td valign="top">

If a node can't be fully matched during the migration of a MIG, the migration finds an alternative match. In the case of segment groups for UN/EDIFACT, the handling has been changed as follows: If `SGnn` of the old message has become `SGmm` in the new message, with the same trigger segment, then segment group `SGnn` is automatically migrated to segment group `SGmm`.
