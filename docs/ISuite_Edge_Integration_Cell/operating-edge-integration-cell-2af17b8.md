# Operating Edge Integration Cell

Learn about the operation procedures for Edge Integration Cell.

You can manage and monitor your Edge Integration Cells with the following functions:

> ### Note:  
> To perform certain operations, you need access to a Kubernetes cluster using the `kubectl` command-line tool.

-   *Operations Cockpit*. Access the central control point for operating your Edge Integration Cells. As an administrator, you can use it to monitor and adjust system configurations and resources.

    For more information, see [Operations Cockpit](operations-cockpit-ec0fc95.md).

-   *System Monitoring*: Access specific monitoring and logging dashboards for your Edge Integration Cell.

    For more information, see [System Monitoring](system-monitoring-689a9a1.md).

-   *Application Monitoring*: Monitor messages and integration artifacts deployed on Edge Integration Cell.

    For more information, see [Application Monitoring](application-monitoring-c9863ba.md).

-   *Log Levels*: Set log levels for Java-based solution components.

    For more information, see [Set Log Levels](set-log-levels-8e58ec4.md).

    [Custom Loggers](custom-loggers-7e52a49.md)

-   *Heap and Thread Dumps:* Create heap and thread dumps for diagnostic analysis.

    For more information, see [Create Heap and Thread Dumps](create-heap-and-thread-dumps-fb10394.md).

    [Diagnostics](diagnostics-80f3050.md)

-   *Message Service Access:* Access the Message Service.

    For more information, see [Access the Message Service](access-the-message-service-abbb36a.md).

-   *Integration Flows with Proxies*: Run integration flows that include manual proxy configuration.

    For more information, see [Execute Integration Flows with Proxies](execute-integration-flows-with-proxies-e9515b5.md).
