# Edge Integration Cell

The Edge Integration Cell is a runtime that enables you to deploy and manage API artifacts within your own private cloud or on-premise environment, supporting hybrid integration scenarios.

As a customer-managed runtime, the Edge Integration Cell allows organizations to host API artifacts closer to their internal systems while still leveraging centralized API management capabilities. This is particularly valuable in scenarios where data residency, latency, compliance, or secure connectivity requirements make it preferable to run APIs within a controlled environment rather than in a fully managed public cloud runtime.

**Key Characteristics**

-   Customer-Managed Runtime: API artifacts run on infrastructure that you own and operate, giving you full control over the underlying environment.
-   Hybrid Deployment Model: API artifacts deployed on the Edge Integration Cell can coexist with those deployed on cloud-based runtimes, enabling flexible integration architectures.
-   Local Processing: API requests and responses are processed within your network boundary, reducing exposure of sensitive data to external systems.
-   Centralized Governance: Although the runtime is local, API artifacts remain governed, monitored, and managed through the central API management platform.

API artifacts deployed on the Edge Integration Cell are designed, configured, and governed through the API management capability on SAP Integration Suite, while their runtime lifecycle—including deployment, scaling, and monitoring—takes place on the Edge Integration Cell within your private infrastructure. For more information, see [Edge Integration Cell](https://help.sap.com/docs/integration-suite/edge-integration-cell-test/what-is-edge-integration-cell?version=CLOUD).
