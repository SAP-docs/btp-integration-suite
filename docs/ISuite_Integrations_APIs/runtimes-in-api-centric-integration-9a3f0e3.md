# Runtimes in API-Centric Integration

API artifacts can be deployed to the Integration Cell for fully managed cloud-based execution, or to the Edge Integration Cell for execution within a customer-managed private cloud or on-premise environment.

In API-centric integration, runtimes are the execution environments where your API artifacts are deployed and run. Choosing the right runtime is essential, as it determines how your APIs are hosted, managed, scaled, and secured. Depending on your business needs—whether you require a fully managed cloud experience or greater control over your infrastructure—you can deploy API artifacts to one of two runtime options.

-   **Integration Cell** – The Integration Cell is a cloud-native runtime purpose-built for designing, deploying, and operating APIs and MCP servers. It delivers enhanced integration capabilities, greater flexibility, and improved operational control—all without the overhead of managing the underlying infrastructure. This option is ideal for organizations seeking a scalable, managed cloud solution. For more information, see [Integration Cell](integration-cell-bd33000.md).
-   **Edge Integration Cell** – The Edge Integration Cell is a customer-managed runtime that can be deployed within your own private cloud or on-premise environment. It is designed to support hybrid integration scenarios, allowing organizations to keep sensitive workloads, data, or connections closer to their internal systems while still leveraging the benefits of API-centric integration. For more information, see [Edge Integration Cell](edge-integration-cell-f8479cb.md).
