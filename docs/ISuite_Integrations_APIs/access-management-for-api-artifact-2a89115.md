# Access Management for API Artifact

Access control for API artifacts enables you to control who can access and manage API artifacts and the data associated with them.

> ### Note:  
> Access control for API artifacts is currently supported only for API artifacts deployed on the *Edge Integration Cell* runtime.

By defining access permissions, you can determine which users or user groups can perform specific actions on an API artifact. For example, users can be granted permissions to view, create, edit, deploy, undeploy, monitor, copy, download, or delete API artifacts. Users who are not assigned the required permissions can be restricted from performing these operations. For more information, see [Managing Access Policies](managing-access-policies-318d107.md).

Access control protects both the API artifact and the data associated with the API artifact. Users who have access to a protected API artifact can also access the data and monitoring information associated with that API artifact.

Access control enables you to manage permissions for:

-   API artifact operations during design time, such as creating, viewing, editing, copying, downloading, and deleting API artifacts.

-   API artifact operations after deployment, such as monitoring the API artifact, changing the log level, and undeploying the API artifact.

-   Access to data and monitoring information associated with a protected API artifact.

For more information, see [Access Policies](access-policies-e0009f3.md) and [Defining Access Policies](defining-access-policies-b0d7950.md).
