# Create an API Artifact by Referring to an API Provider

This method supports API modeling through the discovery of pre-configured business systems or by referencing SAP BTP destinations created for the Integration Cell.



There are two ways to create an API artifact via the API provider method:

-   **Discover APIs from a backend system**: Create an API artifact by referring to a business system, such as:

    -   S/4HANA Cloud — a Unified Customer Landscape \(UCL\)-based business system
    -   SAP Gateway System

    For the configuration steps, see [Creating API Artifact By Referring to a Business Systems](create-an-api-artifact-using-a-discoverable-system-bd5c1f4.md).

-   **Discover an API by referring to an SAP BTP destination**: Create an API artifact by directly referring to a SAP BTP destination that has been created in the subaccount. For the configuration steps, see [Create an API Artifact by Referring to a Destination](create-an-api-artifact-by-referring-to-a-destination-a0bdfd4.md).
