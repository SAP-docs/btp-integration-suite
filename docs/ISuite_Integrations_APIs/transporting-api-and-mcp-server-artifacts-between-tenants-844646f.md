# Transporting API and MCP Server Artifacts Between Tenants

You can reuse API and MCP server artifacts across multiple tenants by transporting them from one tenant \(source\) to another \(target\) via Content Transport Management.

Let us consider an example where you have two tenants – Test and Production. You design and test an API artifact in the Test tenant. Once the API artifact is verified and works as expected, instead of recreating it from scratch in the production tenant, you can simply reuse the tested API artifact by transporting it from test to production.

> ### Note:  
> The steps to configure the system for transporting API and MCP server artifacts are the same as those used for transporting integration content. For more information, see [Content Transport](content-transport-e3c79d6.md).
