# Participant Flow Step

Participant steps enable you to model the remote systems that interact with an API artifact as senders or receivers of messages. You can add these steps to the policy model of your API artifact to represent the external systems participating in message exchange.



## Use

You use participant steps in an API artifact to:

-   Model a remote system as the **sender** of messages to the API.
-   Model a remote system as the **receiver** of messages from the API.
-   Establish clear source and target endpoints for message flow within the API artifact.
