# Integration Cell

Integration Cell extends the runtime capabilities of SAP Integration Suite by offering a fully managed, scalable, and Kubernetes-based environment. It supports unified API modeling, secure execution with built-in traffic management and policy enforcement, and cloud-native deployment with lifecycle governance and monitoring.

This new runtime is part of a broader, flexible SAP Integration Suite landscape, allowing you to choose the most suitable deployment model for your needs. It is a managed, cloud-native runtime for API and MCP server lifecycle management that provides essential services such as security, traffic management, and mediation capabilities. It ensures secure and reliable data exchange between API consumers and backends, facilitating seamless execution of end-to-end integrations.

## API and MCP Server Lifecycle Management on Integration Cell

Integration Cell provides robust capabilities for end-to-end API management:

-   Unified modeling: A common designer for creating and managing APIs and MCP servers, with support for security and traffic management capabilities across both artifact types and mediation capabilities for API artifacts.
-   Enhanced artifact design: Create API and MCP server artifacts that encapsulate backend proxification logic along with security, traffic management, and mediation policies.
-   Policy-Based configuration: Use a visual interface to configure authentication, authorization, rate limiting, IP filtering, and more.
-   Automated discovery: Seamlessly discover APIs and services from your connected landscape using destinations.
-   Lifecycle management: Manage deployment, versioning, and monitoring of APIs and MCP servers with built-in support for ingress and virtual host management.
-   Analytics and monitoring:
    -   API and MCP server logging via Message Processing Logs \(MPL\).
    -   Analytics dashboard for real-time visibility into API and MCP server health, usage metrics, and custom dashboards.

-   Discovery and consumption:
    -   APIs and MCP servers are grouped into products and published to Developer Hub for discovery and consumption by consumers or application developers.
    -   Application developers can self-subscribe to API products via Developer Hub to obtain the necessary security credentials to invoke APIs.
