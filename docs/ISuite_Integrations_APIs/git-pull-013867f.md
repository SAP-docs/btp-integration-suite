# Git Pull

Git Pull is a Git operation used to retrieve and integrate the latest changes from a connected remote Git repository into the local integration artifact, keeping it synchronized with the repository.



## Prerequisites

Ensure that repositories are configured by your tenant administrator. See [Git Access](IntegrationSettings/git-access-ae173b0.md)

Git Pull is available only for artifacts that are already imported from a Git repository.



## Procedure

1.  Choose *Design* \> *Integrations and APIs* to view the list of integration packages.

2.  Select the required integration package and choose *Artifacts* to view the list of artifacts available in the integration package.

3.  On the artifact, choose <span class="SAP-icons-V5"></span> and *Git Pull*.

4.  Check the auto populated artifact details and enter the *PAT Token*.

5.  Choose *Pull*.
