# Discover and Publish Events from Advanced Event Mesh in Developer Hub

As a content administrator, you can discover and publish events from [Advanced Event Mesh](https://help.pubsub.em.services.cloud.sap/Get-Started/get-started-lp.htm) business system on Developer Hub.



<a name="loio188fff20e77d4b0dbee58013a325a686__section_ccw_5tx_gbc"/>

## How to discover events from Advanced Event Mesh in Developer Hub?

To discover the events from Advanced Event Mesh in Developer Hub, you first need to create a destination to establish a connection between Advanced Event Mesh and Developer Hub. See [Create a Destination to Connect Advanced Event Mesh with Developer Hub](create-a-destination-to-connect-advanced-event-mesh-with-developer-hub-cf493ed.md).

Once you’ve successfully established connection, you can view the business system of type **Advanced Event Mesh** on Developer Hub.



<a name="loio188fff20e77d4b0dbee58013a325a686__section_kcb_k5x_gbc"/>

## How do you discover events and publish products from Advanced Event Mesh business system?

You need to have the*AuthGroup.Content.Admin* role collection assigned to you. After you've successfully established connection with Advanced Event Mesh business system, you, as a content administrator, can perform the following actions:

-   Navigate to the *Admin Center* \> *Mange* \> *Content* from the top navigation bar.

-   Fetch the events by selecting the business system of type *Advanced Event Mesh*.

    > ### Note:  
    > The events are located within the Application Domain. An application domain is a predefined category that groups applications by their primary function and contains a list of selectable events relevant to that domain. You can select an application domain to view the events contained within them.

-   To create a product, start by selecting the *Advanced Event Mesh* business system from the *Business Systems* page. Choose the application domain that contain the events. Optionally, you can select the *Show shared events from other domains* checkbox to view shared events from other application domains. Then, choose the event and add it to your product.

    > ### Note:  
    > Each application domain contains a list of events. Events that are **marked** as shared are accessible from all other domains. Events that are **not marked** as shared are accessible only within their own domain.
    > 
    > Events must be shared before they can be added to a product. If an event is not shared, go to the Advanced Event Mesh portal and mark the event as shared.
    > 
    > When creating a product, you can select shared events from your current application domain as well as shared events from other application domains.

    You can easily view and manage the products you have created from the *Products* tab. For more information, see [Discover and Publish Events from Advanced Event Mesh](discover-and-publish-events-from-advanced-event-mesh-f5b9344.md).
